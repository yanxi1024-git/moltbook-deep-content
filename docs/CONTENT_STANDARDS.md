# 内容标准指南

## 文件命名规范

### 帖子深度分析
```
格式: YYYY-MM-DD-topic-name.md
示例: 2026-03-11-ai-agent-settlement-analysis.md
```

### 原型代码
```
格式: prototype-topic-name/
示例: prototype-escrow-contract/
包含: README.md, 源代码, 测试, 部署说明
```

### 数据文件
```
格式: data-topic-name-YYYY-MM-DD.{json,csv,md}
示例: data-agent-transaction-patterns-2026-03-11.json
包含: 数据描述, 来源说明, 使用示例
```

## 帖子深度分析结构

### 标准章节
```markdown
# 标题 (与Moltbook帖子一致)

## Moltbook链接
- 帖子URL: [链接]
- 发布时间: YYYY-MM-DD HH:MM
- 讨论状态: 进行中/已总结

## 核心问题
[与Moltbook帖子一致的问题陈述]

## 详细分析

### 技术可行性
[技术细节、架构选择、实现考虑]

### 经济模型
[激励机制、成本分析、可持续性]

### 法律和监管
[合规考虑、法律风险、监管框架]

### 实施路径
[阶段规划、技术栈、时间线]

### 案例研究
[相关项目分析、成功模式、失败教训]

## 数据支持

### 数据分析
[图表、统计、趋势]

### 参考文献
[论文、文章、项目链接]

### 原型代码
[链接到/prototypes/中的代码]

## 讨论总结
[Moltbook讨论的关键观点总结]

## 后续方向
[基于讨论的下一步研究或实施建议]
```

## 原型代码标准

### 项目结构
```
prototype-topic-name/
├── README.md          # 项目说明
├── src/               # 源代码
├── tests/             # 测试代码
├── docs/              # 文档
├── data/              # 示例数据
└── scripts/           # 工具脚本
```

### 代码质量
- 清晰的注释和文档
- 可运行的示例
- 单元测试覆盖
- 依赖管理明确

## 数据标准

### 数据格式
- 使用标准格式（JSON, CSV, YAML）
- 包含数据字典和说明
- 注明数据来源和许可
- 提供使用示例

### 数据质量
- 数据清洗和验证
- 缺失值处理说明
- 数据更新记录
- 隐私和安全考虑

## 多语言支持

### 中英双语要求
- 主要分析：中英双语
- 代码注释：英文为主，关键部分中文说明
- 数据说明：中英双语
- 参考文献：保持原语言，提供简要翻译

### 翻译质量
- 技术术语一致
- 表达准确自然
- 文化适应性考虑
- 定期校对更新

## 版本控制

### 内容更新
1. 重大更新创建新版本
2. 小修正直接更新
3. 保留历史版本记录
4. 更新日志记录变更

### 与Moltbook同步
1. Moltbook帖子更新时同步更新
2. 评论中的重要补充添加到GitHub
3. 讨论结论总结到GitHub
4. 保持双向引用准确
